## Project Layer Architecture — Tổng quan

Từ `index.dart` exports, ta rút ra **7 layers** của project:

```
lib/
├── common/          ← Config, helpers, hooks, types, utils (foundation)
├── data_source/     ← API clients, database, Firebase, preferences (data)
├── exception/       ← Custom exceptions, mappers, handlers (error handling)
├── model/           ← API models, entities, enums, converters (data models)
├── navigation/      ← Router, guards, observers (routing)
├── resource/        ← Colors, fonts, images, themes (design tokens)
├── ui/              ← Pages, components, popups, base classes (presentation)
├── generated/       ← Codegen output (auto-generated)
├── index.dart       ← Barrel file (auto-generated)
├── di.dart          ← DI setup
├── di.config.dart   ← DI generated config
├── main.dart        ← Entry point
└── app_initializer.dart
```

> 💡 **FE Perspective**
> **Flutter:** Project tổ chức 7 layers theo trách nhiệm: `common/`, `data_source/`, `exception/`, `model/`, `navigation/`, `resource/`, `ui/` — mỗi layer có folder riêng, dependency direction top→bottom.
> **React/Vue tương đương:** `utils/` + `hooks/` ≈ `common/`, `services/` + `api/` ≈ `data_source/`, `types/` ≈ `model/`, `routes/` ≈ `navigation/`, `styles/` ≈ `resource/`, `components/` + `pages/` ≈ `ui/`.
> **Khác biệt quan trọng:** Flutter tách riêng `model/`, `exception/`, `resource/` thành layer độc lập — FE thường gom chung vào `utils/` hoặc `shared/`. Lý do: Flutter app phức tạp hơn (native platform, offline storage).

→ [Mở file gốc](../../base_flutter/lib/di.config.dart)

> 🔎 **Quan sát**
> - **3 registration types** xuất hiện:
>   1. `gh.factory<T>()` — tạo instance **mới** mỗi lần `getIt.get<T>()`
>   2. `gh.factoryAsync<T>()` — factory **async** + `preResolve: true` → resolve **trước khi** DI ready
>   3. `gh.lazySingleton<T>()` — tạo instance **1 lần**, lazy (chỉ tạo khi gọi lần đầu)
> - `factory` dùng cho stateless objects (JSON decoders) — tạo mới mỗi lần, không giữ state
> - `lazySingleton` dùng cho services cần **share state** (API clients, helpers, preferences) — 1 instance toàn app
> - `factoryAsync` + `preResolve` dùng cho platform dependencies cần async initialization (SharedPreferences)

**Pattern chính:**
1. **Barrel file** — 1 `index.dart` export toàn bộ, auto-generated, luôn import qua đây
2. **Layer architecture** — 7 layers rõ ràng: common → data_source → exception → model → navigation → resource → ui
3. **DI registration types** — `factory` (stateless), `lazySingleton` (shared state), `factoryAsync` (async init)
4. **Generated vs manual** — `index.dart` + `di.config.dart` = generated; `di.dart` + `my_app.dart` = manual. Rule: **không sửa file generated**
