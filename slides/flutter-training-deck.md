## ⚠️ The Problem With Traditional Learning

### Cách Học Truyền Thống: Hiệu Quả Với Beginner — Thảm Họa Với Developer Có Kinh Nghiệm

---

Hầu hết các chương trình training đều đi theo cùng một lộ trình:

```
    📖  Lý thuyết (trừu tượng)
          │
          ▼
    👀  Ví dụ minh hoạ (toy project)
          │
          ▼
    📝  Bài tập (tách rời với thực tế)
          │
          ▼
    ✅  Kiểm tra (trên lý thuyết)
```

### Kết quả?

| Vấn đề | Chi tiết |
|---------|----------|
| **Lý thuyết trước, context sau** | Ngồi nghe giảng concept mà chưa thấy nó xuất hiện ở đâu. Kiến thức trôi tuột vì không có điểm neo. |

| **Vào project vẫn bỡ ngỡ** | Dài dòng ngoài lề lan man lý thuyết — mà vào dự án vẫn bỡ ngỡ. |

## 🌱 Pillar 1 — FE → Flutter Translation Layer

### 308+ Điểm Ánh Xạ Trực Tiếp Giữa FE Và Flutter

---

Chương trình này **không dạy lại những khái niệm bạn đã biết**.

Mỗi module đều có callout **"💡 FE Perspective"** — ánh xạ trực tiếp:

| Đã biết (React / Vue) | Sẽ học (Flutter) | Bất ngờ chưa? |
|----------------------|-------------------|----------------|
| `useState()` / `ref()` | `useState()` (Flutter Hooks) | **Cùng tên. Cùng API.** |
| `useEffect([])` | `useEffect([])` (Flutter Hooks) | **API giống hệt.** |
| Redux / Pinia / Zustand | Riverpod (Provider, StateNotifier) | Tư duy giống nhau |
| React Router / Vue Router | `auto_route` (declarative, guards, nesting) | Kiến trúc tương đương |
| Axios interceptors | Dio interceptors | **Gần như 1:1 mapping.** |
| Error Boundaries | `runCatching` + Exception layer | Cùng tư duy |
| CSS Variables | `ThemeData`, `AppColors`, `AppTextStyles` | Cùng concept |
| `localStorage` / cookies | `SharedPreferences`, `flutter_secure_storage` | Flutter version |

---

```
    useState  →  useState      ✅
    useEffect →  useEffect     ✅
    Axios     →  Dio           ✅
    Pinia     →  Riverpod      ✅
```

**Tiết kiệm 60–70% thời gian học** so với chương trình dạy từ đầu.

## 🚩 The Badge System

### 🔴 Phải Tự Viết · 🟡 AI Hỗ Trợ · 🟢 AI Generate

### Biết Phân Loại Mới Là Senior

---

Mỗi concept trong 28 modules đều được phân loại theo **Triple Badge System**:

| Badge | Ý nghĩa | Hành động | Ví dụ |
|-------|---------|-----------|-------|
| 🔴 **MUST-KNOW** | Kiến thức nền tảng, phải hiểu sâu | Tự viết tay, không dùng AI | Widget lifecycle, `BuildContext`, `ref.watch` |
| 🟡 **SHOULD-KNOW** | Quan trọng, AI có thể hỗ trợ | Hiểu → AI assist → review kỹ | Riverpod providers, Dio interceptors, route guards |
| 🟢 **AI-GENERATE** | Boilerplate, codegen, repetitive | AI generate → verify output | Freezed models, injectable config, i18n keys |

---

### Badge Distribution

```
    🔴 MUST-KNOW     ████████████████████░░░░░░  ~40-50%
    🟡 SHOULD-KNOW   ████████████████░░░░░░░░░░  ~30-40%
    🟢 AI-GENERATE    ████████░░░░░░░░░░░░░░░░░░  ~15-25%
```

---

Trong thời đại AI, kỹ năng quan trọng nhất không phải là **"viết mọi thứ bằng tay"** hay **"giao hết cho AI"** —

Mà là **biết phân loại**: cái nào phải hiểu sâu, cái nào delegate được, cái nào chỉ cần verify.

> ### "Đó là kỹ năng của một Senior Developer trong kỷ nguyên AI."

## 📖 Glossary — Technical Terms Reference

Dành cho audience không quen thuộc với các thuật ngữ Flutter:

| Term | Giải thích |
|------|-----------|
| **Riverpod** | State management library — tương đương Redux/Pinia trong React/Vue |
| **Dio** | HTTP client — tương đương Axios |
| **auto_route** | Navigation library — tương đương React Router / Vue Router |
| **Freezed** | Code generation cho immutable models — tương đương TypeScript types + Immer |
| **MVVM** | Model-View-ViewModel — architectural pattern (FE tương đương Redux pattern) |
| **Interceptors** | Middleware chạy trước/sau HTTP request — tương đương Axios interceptors |
| **StateNotifier** | Class quản lý state — tương đương Redux reducer |
| **Provider** | Cơ chế expose state — tương đương React Context / Pinia store |
