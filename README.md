## 🔍 Bài toán

Trong thời đại AI phát triển mạnh mẽ, bối cảnh công nghệ thay đổi nhanh chóng:

- **FE team đã có nền tảng vững:** Các Frontend Developer (React, Vue, Angular) trong team đã khá mạnh so với lĩnh vực của mình — hiểu rõ component architecture, state management, async patterns, clean code.
- **Gò bó tech stack là rủi ro:** Trong kỷ nguyên AI, việc chỉ giỏi một công nghệ duy nhất là hạn chế. Developer cần mở rộng biên giới kỹ năng để thích ứng và tạo ra nhiều giá trị hơn.
- **Tầm nhìn full-cycle:** Công ty hướng tới mục tiêu mỗi thành viên đều là **full-cycle developer** về tech stack lẫn **AI master** — không phải chuyên gia một ngách, mà là engineer đa năng.
- **Training hiện có không phù hợp:** Các khoá Flutter trên thị trường dạy từ zero — giải thích lại component là gì, state management hoạt động thế nào, async/await là gì. FE developer không cần những thứ đó. Họ cần **lớp ánh xạ** (translation layer), không phải khoá beginner.

Hưởng ứng tầm nhìn full-cycle, team FE chủ động lên kế hoạch mở rộng sang mobile — bắt đầu với Flutter — để thêm một thế mạnh mới cho toàn thể thành viên.

**Đây là sáng kiến từ bên trong tổ chức. Khi FE developer đã mạnh, bước tiếp theo tự nhiên là mở rộng — và Flutter là điểm khởi đầu hoàn hảo.**

## 🧭 Cách tiếp cận

Chương trình được xây dựng trên 3 nguyên tắc khác biệt cơ bản so với training Flutter thông thường:

### 1. Tận dụng kiến thức đã có (Leverage)

FE developer đã nắm vững: component lifecycle, state management patterns, async/await, clean architecture, routing, responsive layouts.

Chương trình này **không dạy lại những khái niệm đó**. Thay vào đó, mở rộng năng lực sang mobile bằng cách ánh xạ trực tiếp từ cái đã biết:

| Đã biết (React/Vue) | Sẽ học (Flutter) |
|---|---|
| `useState` / `ref()` | `setState`, `ValueNotifier` |
| Context API / Provide/Inject | `InheritedWidget`, `Provider` |
| Redux / Pinia / Zustand | `BLoC`, `Riverpod` |
| React Router / Vue Router | `GoRouter`, Navigator 2.0 |
| Flexbox / CSS Grid | `Row`, `Column`, `Stack`, `Flex` |
| `useEffect` cleanup | `dispose()`, widget lifecycle |

Ánh xạ này giúp tiết kiệm 60–70% thời gian học so với chương trình dạy từ đầu.

### 2. AI-First Workflow

Năm 2026, developer không cần thuộc hết syntax mới — nhưng phải biết **giao việc cho AI đúng cách**.

Mỗi buổi học đều tích hợp:
- **Task → Keywords & Constraints** — cách prompt AI cho code production-ready
- **Expected Output** — output đúng trông như thế nào
- **Review Checklist** — lỗi AI hay mắc và cách phát hiện
- **Manual Override** — phần nào bắt buộc tự viết tay

Không phải "dùng AI nếu muốn." Đây là workflow có cấu trúc, nơi AI tăng tốc cả việc học lẫn chất lượng output.

### 3. Peer-to-Peer Learning

Không cần trainer bên ngoài. Chương trình thiết kế cho **team tự tổ chức**:

- Mỗi buổi có 1 presenter (rotate)
- Người trình bày phải hiểu sâu hơn để dạy — [hiệu ứng Feynman](https://en.wikipedia.org/wiki/Learning_by_teaching)
- 85 file chuẩn hoá đảm bảo chất lượng nhất quán bất kể ai present
- Chi phí training: **gần bằng 0** (không phí trainer, không subscription)

## 🎯 Kết quả sau 8 tuần

Sau 8 tuần, FE developer sẽ có thêm mobile như một thế mạnh mới:

**Flutter Development**
- Xây dựng ứng dụng Flutter hoàn chỉnh với navigation, state management, networking
- Áp dụng Clean Architecture + Dependency Injection trong dự án thực tế
- Sử dụng thành thạo Riverpod hoặc BLoC cho state management
- Viết unit test, widget test, và integration test
- Profile và tối ưu performance ứng dụng với DevTools
- Tích hợp native platform (camera, storage, permissions)
- Cấu hình CI/CD pipeline và ship lên Play Store & App Store

**AI-Augmented Development**
- Viết prompt có context và constraints để AI gen code 85%+ production-ready
- Review AI-generated code theo production checklist
- Biết khi nào giao cho AI, khi nào tự viết — AI-aware developer
- Dùng AI tools (Copilot, Cursor, Claude) như force multiplier, không phải crutch

**Đánh giá:** Đạt tiêu chuẩn [Middle Mobile Developer](flutter-training/tieu-chuan/middle-level-rubric.md) — tự triển khai feature end-to-end.

## 📅 Lộ trình 16 buổi

| Tuần | Buổi | Chủ đề | Nội dung chính |
|---|---|---|---|
| **1** | 01 | Giới thiệu Dart & Flutter | Dart syntax, types, functions — ánh xạ từ TypeScript |
| | 02 | Dart nâng cao | OOP, generics, async/await, Stream, null safety |
| **2** | 03 | Widget Tree | StatelessWidget, StatefulWidget, BuildContext, lifecycle |
| | 04 | Layout System | Row, Column, Stack, Flex — ánh xạ từ Flexbox/Grid |
| **3** | 05 | Navigation & Routing | Navigator 2.0, GoRouter, deep linking |
| | 06 | State Management cơ bản | setState, InheritedWidget, Provider — ánh xạ từ Context/Vuex |
| **4** | 07 | Riverpod | Provider types, ref, family, autoDispose |
| | 08 | BLoC Pattern | Cubit, Bloc, Event/State, BlocBuilder |
| **5** | 09 | Clean Architecture | Layer separation, domain/data/presentation, use cases |
| | 10 | DI & Testing | get_it, injectable, unit test, widget test, mockito |
| **6** | 11 | Networking | Dio, Retrofit, interceptors, error handling |
| | 12 | Local Storage | SharedPreferences, Hive, SQLite, secure storage |
| **7** | 13 | Performance | DevTools, profiling, const widgets, lazy loading |
| | 14 | Animation | Implicit/Explicit animation, Hero, Lottie, Rive |
| **8** | 15 | Platform Integration | Method channels, camera, permissions |
| | 16 | CI/CD & Production | Flavors, Fastlane, GitHub Actions, store deployment |

Mỗi buổi gồm **5 file chuẩn hoá**: overview, lý thuyết (có ánh xạ FE↔Flutter), code ví dụ, bài tập thực hành, và tài liệu tham khảo.
